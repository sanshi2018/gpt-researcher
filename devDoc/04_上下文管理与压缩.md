## 4.2 ContextManager — 统一上下文接口

**源码位置**：`gpt_researcher/skills/context_manager.py`

### 三个核心方法

```python
class ContextManager:
    def __init__(self, researcher):
        self.researcher = researcher
    
    # 方法1: 从抓取的页面中获取相关上下文
    async def get_similar_content_by_query(self, query: str, pages: list) -> str:
        context_compressor = ContextCompressor(
            documents=pages,
            embeddings=self.researcher.memory.get_embeddings(),
            prompt_family=self.researcher.prompt_family,
        )
        return await context_compressor.async_get_context(
            query=query, max_results=10, cost_callback=self.researcher.add_costs
        )
    
    # 方法2: 从向量库获取相关上下文
    async def get_similar_content_by_query_with_vectorstore(self, query, filter):
        vectorstore_compressor = VectorstoreCompressor(
            self.researcher.vector_store, filter=filter, ...
        )
        return await vectorstore_compressor.async_get_context(query=query, max_results=8)
    
    # 方法3: 查找已写内容中的相关章节（避免重复写作）
    async def get_similar_written_contents_by_draft_section_titles(
        self, current_subtopic, draft_section_titles, written_contents, max_results=10
    ):
        all_queries = [current_subtopic] + draft_section_titles
        results = await asyncio.gather(*[process_query(q) for q in all_queries])
        return list(set().union(*results))[:max_results]
```

### 方法3的巧妙设计

用于 Detailed Report 生成时，避免不同子主题的内容重复：

```
子主题A的章节标题 ──→ 与已写内容做相似度匹配 ──→ 返回相关内容
    ↓
子主题A的报告生成时，LLM 会看到这些相关内容
    ↓
提示词要求 LLM "不要重复这些已写内容"
```

## 4.3 ContextCompressor — 核心压缩管道

**源码位置**：`gpt_researcher/context/compression.py`

### 压缩管道架构

```
原始文档 (抓取的网页内容)
    ↓
┌── 判断是否需要压缩 ──┐
│                       │
│  total_chars < 8000   │  total_chars >= 8000
│  AND docs <= max_results   │
│                       │
├── 快速通道 ────────┐  ├── 标准管道 ──────────────┐
│  直接返回文档      │  │                            │
│  (跳过 Embedding)  │  │  1. RecursiveCharTextSplit  │
│                    │  │     chunk_size=1000         │
│                    │  │     chunk_overlap=100       │
│                    │  │                            │
│                    │  │  2. EmbeddingsFilter        │
│                    │  │     similarity_threshold    │
│                    │  │     = 0.35 (默认)           │
│                    │  │                            │
│                    │  │  3. ContextualCompression   │
│                    │  │     Retriever               │
│                    │  │                            │
└────────────────────┘  └────────────────────────────┘
         ↓                         ↓
    pretty_print_docs()     pretty_print_docs()
         ↓                         ↓
    格式化上下文字符串
```

### 核心代码

```python
class ContextCompressor:
    def __init__(self, documents, embeddings, max_results=5, ...):
        self.documents = documents
        self.embeddings = embeddings
        self.similarity_threshold = os.environ.get("SIMILARITY_THRESHOLD", 0.35)
    
    async def async_get_context(self, query, max_results=5, cost_callback=None):
        # === 优化: 小文档直接返回 ===
        total_chars = sum(len(str(doc.get('raw_content', ''))) for doc in self.documents)
        chunk_threshold = int(os.environ.get("COMPRESSION_THRESHOLD", "8000"))
        
        if total_chars < chunk_threshold and len(self.documents) <= max_results:
            # 快速通道: 无需压缩
            direct_docs = [
                Document(page_content=doc.get('raw_content', ''), metadata=doc)
                for doc in self.documents[:max_results]
            ]
            return self.prompt_family.pretty_print_docs(direct_docs, max_results)
        
        # === 标准压缩管道 ===
        compressed_docs = self.__get_contextual_retriever()
        if cost_callback:
            cost_callback(estimate_embedding_cost(...))
        relevant_docs = await asyncio.to_thread(compressed_docs.invoke, query)
        return self.prompt_family.pretty_print_docs(relevant_docs, max_results)
```

### 压缩管道构建

```python
def __get_contextual_retriever(self):
    # Step 1: 文本分块器
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=1000, 
        chunk_overlap=100
    )
    
    # Step 2: Embedding 相似度过滤器
    relevance_filter = EmbeddingsFilter(
        embeddings=self.embeddings,
        similarity_threshold=self.similarity_threshold  # 0.35
    )
    
    # Step 3: 组装压缩管道
    pipeline_compressor = DocumentCompressorPipeline(
        transformers=[splitter, relevance_filter]
    )
    
    # Step 4: 包装为检索器
    base_retriever = SearchAPIRetriever(pages=self.documents)
    contextual_retriever = ContextualCompressionRetriever(
        base_compressor=pipeline_compressor, 
        base_retriever=base_retriever
    )
    
    return contextual_retriever
```

### 关键参数

| 参数 | 默认值 | 环境变量 | 说明 |
|------|--------|---------|------|
| `chunk_size` | 1000 | - | 分块大小（字符数） |
| `chunk_overlap` | 100 | - | 分块重叠 |
| `similarity_threshold` | 0.35 | `SIMILARITY_THRESHOLD` | 最低相似度阈值 |
| `compression_threshold` | 8000 | `COMPRESSION_THRESHOLD` | 跳过压缩的阈值 |
| `max_results` | 10 | - | 最大返回文档数 |

## 4.4 SearchAPIRetriever — 基础检索器

**源码位置**：`gpt_researcher/context/retriever.py`

### 两种检索器

```python
class SearchAPIRetriever(BaseRetriever):
    """将 GPT-Researcher 的页面数据转为 LangChain Document"""
    pages: List[Dict] = []
    
    def _get_relevant_documents(self, query, ...):
        return [
            Document(
                page_content=page.get("raw_content", ""),
                metadata={
                    "title": page.get("title", ""),
                    "source": page.get("url", ""),
                },
            )
            for page in self.pages
        ]

class SectionRetriever(BaseRetriever):
    """用于已写章节的检索，保留 section_title 元数据"""
    sections: List[Dict] = []
    
    def _get_relevant_documents(self, query, ...):
        return [
            Document(
                page_content=page.get("written_content", ""),
                metadata={"section_title": page.get("section_title", "")},
            )
            for page in self.sections
        ]
```

### 设计目的

这两个 Retriever 并不实际"检索"，而是 **适配器**：
- `SearchAPIRetriever`：将抓取的网页内容包装为 LangChain Document
- `SectionRetriever`：将已写的报告章节包装为 LangChain Document

它们的存在是为了适配 `ContextualCompressionRetriever` 的接口要求。

## 4.6 WrittenContentCompressor — 已写内容去重

```python
class WrittenContentCompressor:
    """在 Detailed Report 中避免子主题间内容重复"""
    
    def __init__(self, documents, embeddings, similarity_threshold=0.5):
        # 注意: 阈值为 0.5，比普通压缩的 0.35 更高
        self.similarity_threshold = similarity_threshold
    
    async def async_get_context(self, query, max_results=5):
        compressed_docs = self.__get_contextual_retriever()
        relevant_docs = await asyncio.to_thread(compressed_docs.invoke, query)
        return self.__pretty_docs_list(relevant_docs, max_results)
    
    def __pretty_docs_list(self, docs, top_n):
        """返回包含 section_title 的格式化列表"""
        return [
            f"Title: {d.metadata.get('section_title')}\nContent: {d.page_content}\n"
            for i, d in enumerate(docs) if i < top_n
        ]
```

## 4.7 上下文输出格式化

```python
# prompts.py — pretty_print_docs()
@staticmethod
def pretty_print_docs(docs, top_n=None):
    """将 Document 列表格式化为上下文字符串"""
    return "\n".join(
        f"Source: {d.metadata.get('source')}\n"
        f"Title: {d.metadata.get('title')}\n"
        f"Content: {d.page_content}\n"
        for i, d in enumerate(docs)
        if top_n is None or i < top_n
    )
```

输出示例：
```
Source: https://example.com/article1
Title: Understanding RAG
Content: RAG (Retrieval Augmented Generation) is a technique...

Source: https://example.com/article2
Title: RAG vs Fine-tuning
Content: When comparing RAG with fine-tuning...
```

## 4.8 SourceCurator — LLM 驱动的来源筛选

**源码位置**：`gpt_researcher/skills/curator.py`

当 `cfg.curate_sources = True` 时，使用 LLM 评估来源质量：

```python
class SourceCurator:
    async def curate_sources(self, source_data, max_results=10):
        response = await create_chat_completion(
            model=self.researcher.cfg.smart_llm_model,
            messages=[
                {"role": "system", "content": self.researcher.role},
                {"role": "user", "content": self.researcher.prompt_family.curate_sources(
                    self.researcher.query, source_data, max_results
                )},
            ],
            temperature=0.2,
            max_tokens=8000,
        )
        return json.loads(response)
```

### 筛选标准（来自 Prompt）

1. **相关性**：与研究查询的关联度
2. **可信度**：来源权威性
3. **时效性**：偏好新数据
4. **客观性**：即使有偏见也保留独特视角
5. **数据价值**：优先包含统计数据和具体数字的来源

## 4.9 Deep Research 中的上下文裁剪

**源码位置**：`gpt_researcher/skills/deep_research.py`

Deep Research 产生大量上下文，需要裁剪以避免超出 LLM 上下文窗口：

```python
MAX_CONTEXT_WORDS = 25000  # 25k 词安全边界

def trim_context_to_word_limit(context_list, max_words=MAX_CONTEXT_WORDS):
    """裁剪上下文列表，保留最近/最相关的条目"""
    total_words = 0
    trimmed_context = []
    
    # 从后往前处理 → 保留最新的条目
    for item in reversed(context_list):
        words = count_words(item)
        if total_words + words <= max_words:
            trimmed_context.insert(0, item)  # 插入开头保持原始顺序
            total_words += words
        else:
            break
    
    return trimmed_context
```

### 裁剪策略

- **最新优先**：假设最近添加的上下文更相关（来自更深层递归）
- **25k 词限制**：为 LLM 留出足够的生成空间
- **保持顺序**：虽然从后往前选择，但最终保持原始顺序
