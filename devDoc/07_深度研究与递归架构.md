## 7.1 Deep Research 概念

Deep Research 是 GPT-Researcher 中最复杂的功能，实现了 **树状递归研究**：

```
             [原始查询]
            /    |    \       ← 广度 (breadth=4)
        [子查询1] [子查询2] [子查询3] [子查询4]
        /   \     /   \     /   \     /   \    ← 深度 (depth=2)
     [孙1] [孙2] [孙3] [孙4] ...             
```

每一层都执行完整的 `GPTResearcher.conduct_research()` 流程，然后将结果向上聚合。

### 核心数据结构

```python
class ResearchProgress:
    """追踪递归研究进度"""
    total_research_tasks: int = 0    # 总任务数
    completed_tasks: int = 0          # 已完成数
    total_depth: int = 0              # 配置的最大深度
    total_breadth: int = 0            # 配置的广度

class DeepResearchSkill:
    def __init__(self, researcher):
        self.researcher = researcher
        
        # 从配置读取参数

        self.depth = cfg.deep_research_depth or 2           # 最大递归深度
        self.concurrency = cfg.deep_research_concurrency or 2  # 最大并发数
        
        # 共享资源
        self.visited_urls = researcher.visited_urls           # URL去重
        self.learnings = []                                    # 累积的学习成果
        self.sources = []                                      # 累积的来源
        
        # 并发控制
        self.semaphore = asyncio.Semaphore(self.concurrency)  # 信号量限制并发
```

## 7.3 deep_research() — 主递归方法

```python
async def deep_research(
    self,
    query: str,
    breadcrumbs: str = "",    # 面包屑路径
    depth: int = 0,           # 当前深度
    learnings: list = None,   # 累积学习
    progress: ResearchProgress = None,
):
    """树状递归研究的核心方法"""
    
    # === 递归终止条件 ===
    if depth >= self.depth:
        return {
            "learnings": learnings,
            "visited_urls": list(self.visited_urls),
        }
    
    # === Step 1: 生成 SERP 查询（带上下文的子查询） ===
    serp_queries = await self._generate_serp_queries(
        query=query,
        num_queries=self.breadth,
        learnings=learnings,      # 传入已有学习，避免重复
    )
    
    # === Step 2: 并发执行子查询（受信号量限制） ===
    tasks = []
    for serp_query in serp_queries:
        async def process_query(sq):
            async with self.semaphore:  # 并发控制 ⚡
                return await self._process_single_query(
                    serp_query=sq,
                    query=query,
                    breadcrumbs=breadcrumbs,
                    depth=depth,
                    learnings=learnings,
                    progress=progress,
                )
        tasks.append(process_query(serp_query))
    
    # === Step 3: 并行执行并收集结果 ===
    results = await asyncio.gather(*tasks, return_exceptions=True)
    
    # === Step 4: 合并学习成果 ===
    for result in results:
        if isinstance(result, dict):
            new_learnings = result.get("learnings", [])
            learnings = list(set(learnings + new_learnings))
    
    return {
        "learnings": learnings,
        "visited_urls": list(self.visited_urls),
    }
```

## 7.4 _process_single_query() — 单查询处理

```python
async def _process_single_query(self, serp_query, query, breadcrumbs, 
                                  depth, learnings, progress):
    """处理单个子查询 — 包含递归调用"""
    
    query_text = serp_query['query']
    research_goal = serp_query.get('research_goal', '')
    
    # === Step 1: 创建子 Researcher 实例 ===
    researcher = GPTResearcher(
        query=query_text,
        report_type="deep_research",
        parent_query=query,             # 父查询上下文
        visited_urls=self.visited_urls, # 共享已访问URL ⭐
        verbose=self.researcher.verbose,
        ...
    )
    
    # === Step 2: 执行研究 ===
    await researcher.conduct_research()
    
    # === Step 3: 提取学习成果 ===
    report = await researcher.write_report()
    new_learnings = self._extract_learnings(report, query_text)
    
    # === Step 4: 更新进度 ===
    progress.completed_tasks += 1
    await self._report_progress(progress)
    
    # === Step 5: ⭐ 递归——对子查询继续深入 ===
    new_breadcrumbs = f"{breadcrumbs} > {query_text}"
    deeper_result = await self.deep_research(
        query=query_text,
        breadcrumbs=new_breadcrumbs,    # 面包屑追踪路径
        depth=depth + 1,                # 深度+1
        learnings=learnings + new_learnings,
        progress=progress,
    )
    
    return {
        "learnings": deeper_result.get("learnings", []),
        "visited_urls": deeper_result.get("visited_urls", []),
    }
```

## 7.5 _generate_serp_queries() — 智能子查询生成

```python
async def _generate_serp_queries(self, query, num_queries, learnings):
    """生成 SERP 查询，考虑已有学习避免重复"""
    
    # 将已有学习转为摘要，提供给 LLM
    learnings_summary = "\n".join([
        f"- {learning}" for learning in learnings[-20:]  # 最近20条
    ])
    
    prompt = f"""Given the research query: "{query}"
    
    And the following learnings from previous research:
    {learnings_summary}
    
    Generate {num_queries} NEW search queries that:
    1. Explore different angles of the topic
    2. Do NOT repeat already-covered information
    3. Go deeper into underexplored areas
    4. Are specific enough to yield relevant results
    
    Return JSON: [
        {{"query": "search query", "research_goal": "what we want to learn"}},
        ...
    ]
    """
    
    response = await create_chat_completion(
        model=self.researcher.cfg.strategic_llm_model,
        messages=[{"role": "user", "content": prompt}],
        ...
    )
    
    return json_repair.loads(response)
```

### 关键设计：避免重复研究

每次生成子查询时，都将已有学习作为上下文传入，确保：
1. 不重复已探索的方向
2. 聚焦于信息缺口
3. 逐层深入细节

## 7.6 并发控制机制

### Semaphore 信号量

```python
# 初始化时设置最大并发数
self.semaphore = asyncio.Semaphore(self.concurrency)  # 默认 2

# 使用时通过 async with 自动管理
async with self.semaphore:
    result = await self._process_single_query(...)
```

### 为什么需要并发控制？

```
depth=2, breadth=4 的树状结构:

深度0: 1 个查询
深度1: 4 个查询 (并行)
深度2: 4×4 = 16 个查询 (并行)

总计: 1 + 4 + 16 = 21 个 GPTResearcher 实例
每个实例都要调用 LLM + 搜索API + 抓取网页

如果不限制并发:
- API 费率限制 → 大量 429 错误
- 内存压力 → 21 个 Researcher 同时运行
- 网络拥塞 → 抓取超时
```

    ...

### 上下文裁剪

```python
MAX_CONTEXT_WORDS = 25000

def trim_context_to_word_limit(context_list, max_words=MAX_CONTEXT_WORDS):
    """保留最新的学习成果，限制总词数"""
    total_words = 0
    trimmed_context = []
    
    for item in reversed(context_list):
        words = count_words(item)
        if total_words + words <= max_words:
            trimmed_context.insert(0, item)
            total_words += words
        else:
            break
    
    return trimmed_context
```

### 聚合流程

```
深度2 结果 → 向上传递 learnings
深度1 结果 + 深度2 learnings → 合并去重
深度0 → 最终 learnings 列表

去重: learnings = list(set(learnings + new_learnings))

最终报告生成时:
  context = trim_context_to_word_limit(all_learnings, 25000)
  report = generate_deep_research_prompt(query, context, ...)
```

## 7.8 进度追踪与报告

```python
class ResearchProgress:
    total_research_tasks: int = 0
    completed_tasks: int = 0
    total_depth: int = 0
    total_breadth: int = 0
    current_depth: int = 0

async def _report_progress(self, progress):
    """通过 WebSocket 推送进度"""
    if self.researcher.websocket:
        percentage = (progress.completed_tasks / progress.total_research_tasks) * 100
        await stream_output(
            "logs", "deep_research_progress",
            f"📊 Deep Research Progress: {percentage:.0f}% "
            f"({progress.completed_tasks}/{progress.total_research_tasks} tasks, "
            f"Depth: {progress.current_depth}/{progress.total_depth})",
            self.researcher.websocket
        )
```

### 任务总数计算

```python
# 总任务数 = sum(breadth^i for i in range(depth+1)) - 1
# 例: breadth=4, depth=2
#   = 4^0 + 4^1 + 4^2 - 1 = 1 + 4 + 16 - 1 = 20
```

## 7.9 Deep Research 专用 Prompt

```python
# prompts.py — generate_deep_research_prompt()
def generate_deep_research_prompt(question, context, report_source, ...):
    return f"""
Using the following hierarchically researched information and citations:

"{context}"

Write a comprehensive research report answering the query: "{question}"

The report should:
1. Synthesize information from multiple levels of research depth
2. Integrate findings from various research branches
3. Present a coherent narrative that builds from foundational to advanced insights
4. Maintain proper citation of sources throughout
5. Be well-structured with clear sections and subsections
6. Have a minimum length of {total_words} words

Additional requirements:
- Prioritize insights that emerged from deeper levels of research
- Highlight connections between different research branches
- Include relevant statistics, data, and concrete examples
"""
```

### 与普通报告 Prompt 的区别

| 维度 | 普通报告 | 深度研究报告 |
|------|---------|-------------|
| 上下文来源 | 单次搜索 | 多层递归研究 |
| 特殊指令 | 无 | "Synthesize from multiple levels" |
| 引用处理 | 标准 | "Maintain proper citation throughout" |
| 洞察优先级 | 无 | "Prioritize deeper-level insights" |
| 最小字数 | 1000 | 2000 |
