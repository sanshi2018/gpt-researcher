# 模块五：短时记忆与上下文防爆 (Memory & Context Control)

如果你玩过最早期的 ChatGPT，一定遇到过这个问题：聊着聊着，它就把最开始的设定给忘了。或者，当你往对话框里塞入了几万字的文档，系统直接报错：“超出 Token 限制 (Max Tokens Exceeded)”。

在一个自动化的 Agent 系统中，这两大问题尤为致命。如果 Agent 陷入死循环，疯狂去爬同一个网站；或者把 20 万字的数据直接扔给 LLM 导致 API 报错，整个任务就会直接崩溃。

今天，我们就来解剖 `gpt-researcher` 是如何通过**记忆管理**与**适配器模式**，防患于未然的。

---

## 1. 短期防重记忆：`visited_urls` 机制

Agent 在发散搜索时，不同的搜索词（Sub-queries）往往会搜出相同的网页。比如搜“iPhone 15 评测”和“苹果最新手机 体验”，大概率都会搜到少数派或知乎的同一篇文章。

如果在代码里不加干预，Agent 会傻乎乎地把这个网页爬两次、存两次，浪费双倍的时间和金钱。

打开 `gpt_researcher/skills/researcher.py`，你会看到一个无处不在的变量：`self.researcher.visited_urls`。

```python
# gpt_researcher/skills/researcher.py 节选
async def _get_new_urls(self, url_set_input):
    new_urls = []
    for url in url_set_input:
        # 核心防重逻辑：如果这个网址没去过，才加进去
        if url not in self.researcher.visited_urls:
            self.researcher.visited_urls.add(url)
            new_urls.append(url)
            # ... 打印日志 "✅ Added source url to research: {url}"
    return new_urls
```

**看点**：这其实就是计算机基础数据结构中 `Set` (集合) 的最经典应用。利用 Set 原生自带的去重特性，GPT-Researcher 为 Agent 赋予了“短期记忆”，不仅避免了重复爬取，最后在生成报告底部**参考文献 (References)** 的时候，也是直接把这个 Set 打印出来。

## 2. 上下文压缩的基石：`memory/embeddings.py`

我们在模块四中讲过，解决 LLM 被撑爆的终极方案是 **RAG 上下文压缩**。而支撑这个压缩过程的核心底层依赖，是**向量模型 (Embedding Model)**。

GPT-Researcher 是一个完全开源、支持私有化部署的项目。这意味着用户可能没有 OpenAI，只能用本地的 Ollama，或者国产的阿里 DashScope。

如何优雅地支持多达 20 种不同的底层模型？它使用了非常经典的**适配器模式 (Adapter Pattern)**。

打开 `gpt_researcher/memory/embeddings.py` 中的 `Memory` 类：

```python
# gpt_researcher/memory/embeddings.py 节选
class Memory:
    def __init__(self, embedding_provider: str, model: str, **embedding_kwargs):
        _embeddings = None
        match embedding_provider:
            case "openai":
                from langchain_openai import OpenAIEmbeddings
                _embeddings = OpenAIEmbeddings(model=model, **embedding_kwargs)
            case "ollama":
                from langchain_ollama import OllamaEmbeddings
                _embeddings = OllamaEmbeddings(
                    model=model, 
                    base_url=os.environ["OLLAMA_BASE_URL"]
                )
            case "google_genai":
                from langchain_google_genai import GoogleGenerativeAIEmbeddings
                _embeddings = GoogleGenerativeAIEmbeddings(...)
            # ... 省略几十个其他厂商
            case _:
                raise Exception("Embedding not found.")

        self._embeddings = _embeddings

    def get_embeddings(self):
        return self._embeddings
```

**看点**：
1. **延迟加载 (Lazy Loading)**：仔细看 `import` 语句，它是写在 `case` 分支里面的！这意味着如果你只用 OpenAI，程序根本不会去加载 `langchain_ollama` 等依赖库。极大降低了运行时的内存开销。
2. **统一接口**：外部调用时，根本不需要关心用的是什么模型，只要调用 `memory.get_embeddings()` 就能无缝对接前一个模块讲到的 `ContextCompressor` 压缩器。

## 3. 实操体验：本地化 Embedding 记忆池 (MRE)

让我们写一个无需大模型 API 也能运行的脚本。我们将模拟 Agent 去重和使用本地化模型（通过工厂类 `Memory`）进行初始化的过程。

*(注：如果你没有配置其他模型，我们将默认使用 OpenAI；如果你有 Ollama，你可以将其改为 `ollama`)*

### MRE 代码：Agent 记忆控制器

在根目录下新建文件 `mre_module5.py`：

```python
import os
from gpt_researcher.memory.embeddings import Memory

def main():
    print("🧠 Agent 记忆机制测试启动\n")

    # --- 1. 短期防重记忆测试 ---
    print("【测试 1: Visited URLs 集合去重】")
    visited_urls = set()
    
    # 模拟 Agent 在三次搜索中找到的链接
    search_1 = ["https://arxiv.org/1", "https://news.com/a"]
    search_2 = ["https://news.com/a", "https://wiki.org/b"] # news.com/a 重复了！
    
    for url in search_1 + search_2:
        if url not in visited_urls:
            visited_urls.add(url)
            print(f"  ✅ 新发现并加入记忆库: {url}")
        else:
            print(f"  ❌ 已经访问过了，跳过: {url}")
            
    print(f"最终待爬取队列共 {len(visited_urls)} 个链接。\n")


    # --- 2. 向量记忆池初始化测试 ---
    print("【测试 2: 多态 Memory 适配器】")
    # 假设用户的配置文件里写的是 openai
    provider = "openai"
    model_name = "text-embedding-3-small"
    
    # 检查环境变量避免报错
    if not os.environ.get("OPENAI_API_KEY"):
        print("⚠️ 警告: 未找到 OPENAI_API_KEY，将使用一个伪造的 Key 进行测试")
        os.environ["OPENAI_API_KEY"] = "sk-dummy-key"
        
    try:
        # 实例化记忆池
        memory_pool = Memory(embedding_provider=provider, model=model_name)
        embeddings_instance = memory_pool.get_embeddings()
        
        print(f"✅ 成功加载 Embedding 模型！")
        print(f"当前使用的底层类: {embeddings_instance.__class__.__name__}")
        
    except Exception as e:
        print(f"❌ 加载失败: {e}")

if __name__ == "__main__":
    main()
```

### 运行结果预期
运行这个脚本（`python mre_module5.py`），你会看到：
1. 第一个测试中，程序精准捕捉到了重复的 `news.com/a` 并拒绝再次加入任务队列。这正是 Agent 保证不出错、不烧钱的底线。
2. 第二个测试中，`Memory` 类像个聪明的管家，根据字符串 `openai`，默默在后台帮你把 `OpenAIEmbeddings` 类实例化完毕，等待被用作文本降维的武器。
