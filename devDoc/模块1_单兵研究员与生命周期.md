# 模块一：单兵研究员与生命周期 (The Core Researcher Engine)

欢迎来到我们的第一个实战模块。在这个模块中，我们将拆解 `gpt-researcher` 最核心的引擎文件：`gpt_researcher/agent.py`，并带你从头到尾走通一个“单兵研究员”的完整生命周期。

---

## 1. 原理剖析：GPTResearcher 究竟是个啥？

在多智能体 (Multi-Agent) 大行其道的今天，很多人以为 `gpt-researcher` 一上来就是一群机器人在开会。其实并非如此。它的最底层基石，是一个极为强大、全能的**“单兵研究员”（Single Agent）**，即 `GPTResearcher` 类。

你可以把 `GPTResearcher` 想象成一家新闻社的**“全能主编”**。他虽然挂着主编的头衔，但底下其实管着一堆**“部门经理”**：
*   **ResearchConductor (调查经理)**：负责带人出去跑外勤，去 Google 搜网页、去爬取内容。
*   **ContextManager (档案经理)**：负责把拿回来的几万字杂乱网页，精简提炼成最关键的事实。
*   **BrowserManager (爬虫经理)**：负责具体的网页抓取脏活累活。
*   **ReportGenerator (主笔撰稿人)**：负责拿着最终提炼好的事实，写出 Markdown 格式的漂亮长文。

这就是面向对象编程在 Agent 设计中的绝佳体现：**主逻辑极其干净，脏活累活全部下沉。**

---

## 2. 深层解剖：核心代码流转

让我们深入到 `gpt_researcher/agent.py` 中，看看这位主编是如何度过他的一生的（生命周期）：

### 节点 1：接单与初始化 (`__init__`)
当你实例化一个 `GPTResearcher` 时，你必须传入 `query`（用户的研究问题）。
在这里，它会把所有的部门经理都给初始化好：
```python
# gpt_researcher/agent.py 截取
self.research_conductor = ResearchConductor(self)
self.report_generator = ReportGenerator(self)
self.context_manager = ContextManager(self)
self.scraper_manager = BrowserManager(self)
```
注意这里的设计模式：主编 `self` 被当做参数传给了各个经理。这样，所有的经理都能共享同一个配置文件 `cfg` 和记忆池 `memory`。

### 节点 2：角色分配与出外勤 (`conduct_research`)
一旦调用 `await researcher.conduct_research()`，大戏正式开场。
1.  **“千人千面”的动态角色分配**：主编会先调用 `choose_agent()`。如果用户问的是“马斯克的火箭”，主编会通过 LLM 判断，临时给自己套上一个**“资深航天领域分析师”**的 System Prompt（人设）。这决定了后续查资料和写文章的语气。
2.  **出外勤查资料**：接着调用 `self.research_conductor.conduct_research()`。此时，调查经理开始工作，生成子查询（Sub-queries）、爬网页、提炼上下文。
3.  **结果返回**：外勤跑完后，所有的有效事实都会被保存在 `self.context` 里。

### 节点 3：撰写定稿 (`write_report`)
拿到了 `context` 后，调用 `await researcher.write_report()`。
此时，**主笔撰稿人** (`ReportGenerator`) 接手，他会根据一开始分配的“航天分析师”人设，结合 `context` 中的干货，按照 Markdown 的排版要求，输出最终的万字研报。

## 3. 实操体验：最小可运行示例 (MRE)

看源码不如直接跑跑看。下面我为你准备了一个极其轻量级的 MRE。我们不需要启动复杂的前端和 FastAPI 服务器，只需短短十几行代码，就能在本地见证这位单兵研究员的威力。

### 环境准备
1. 确保你在 `gpt-researcher` 根目录下，并安装了依赖。
2. 配置好你的 `.env` 文件。核心需要：
    *   `OPENAI_API_KEY`（或其他你配置的 LLM Key，如 `ANTHROPIC_API_KEY`, `GEMINI_API_KEY`）
    *   `TAVILY_API_KEY`（搜索引擎，推荐去 tavily.com 免费注册一个）

### MRE 代码：单兵研究员的初体验
请在 `gpt-researcher` 的根目录下，新建一个文件叫 `mre_module1.py`，并填入以下代码：

```python
# mre_module1.py
import asyncio
from gpt_researcher import GPTResearcher

async def main():
    # 1. 设定研究任务 (Query)
    query = "总结 2024 年 Apple Vision Pro 的市场表现与核心槽点"
    print(f"🚀 任务发布: {query}")

    # 2. 实例化全能主编 (GPTResearcher)
    # 我们指定报告类型为 research_report (标准研报)
    researcher = GPTResearcher(
        query=query, 
        report_type="research_report"
    )

    # 3. 让主编带人出外勤，收集核心事实 (Conduct Research)
    print("\n🔍 正在满世界搜集情报，这可能需要一两分钟，请耐心等待...")
    research_context = await researcher.conduct_research()
    print("\n✅ 情报收集完毕！截取前 500 个字符看看抓到了什么：")
    print("-" * 50)
    print(research_context[:500] + "......")
    print("-" * 50)

    # 4. 让主笔撰稿人开始写报告 (Write Report)
    print("\n✍️ 正在根据收集到的情报撰写深度研报...")
    report = await researcher.write_report()
    
    # 5. 打印最终成果
    print("\n🎉 报告撰写完毕：\n")
    print(report)

if __name__ == "__main__":
    # 运行异步主函数
    asyncio.run(main())
```

### 如何运行与观察
打开终端，运行 `python mre_module1.py`。
在运行过程中，请仔细观察终端输出的日志，你会隐约看到它在后台自动生成了诸如 `"Apple Vision Pro 销量 2024"`, `"Apple Vision Pro 用户评价 痛点"` 等多个子搜索词。这就是这位“单兵研究员”在自主思考！
