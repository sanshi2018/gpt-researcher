# 模块六：万字长文组装车间 (Report Generation & Assembly)

到目前为止，我们的 Agent 已经完成了所有“搬砖”的工作：找线索、爬网页、读长文、做笔记（Context 压缩）。
现在，到了最考验“主编”笔杆子的时候了——**如何把这些散碎的笔记，变成一篇几万字的结构化深度研报？**

如果你曾让 ChatGPT 帮你写一万字的文章，你一定知道它根本做不到。它不仅会中途断掉（超出输出 Token 限制），还会越写越水，最后前后矛盾。

GPT-Researcher 解决这个问题的思路，堪称**大模型长文本生成的教科书**。

---

## 1. 核心机制：化整为零的 Map-Reduce 范式

GPT-Researcher 绝对不会让大模型“一口吃成个胖子”。它采用了一种分步骤、流水线式的车间作业模式：

1. **拟定大纲 (Plan)**：先根据总目标，让大模型生成一份包含几个核心章节的目录。
2. **分包撰写 (Map)**：对于目录中的**每一个小节 (Subtopic)**，系统会单独开启一个 LLM 线程，带着专属于这个小节的上下文 (Context)，让它只写这几百字。
3. **统筹首尾 (Head & Tail)**：当所有中间章节都写完后，再把它们合并，让 LLM 统览全文，单独写出“引言 (Introduction)”和“结论 (Conclusion)”。
4. **无缝拼接 (Reduce)**：最后，将引言、各个小节、结论、以及我们前面提到的**参考文献 (Visited URLs)**，用 Markdown 格式拼装在一起。

## 2. 源码深层解剖：`actions/report_generation.py`

打开 `gpt_researcher/actions/report_generation.py`，这就是整个拼装车间的主控室。

### 阶段一：大纲与分块生成
注意看这里定义了非常明确的颗粒度函数：
*   `generate_draft_section_titles()`：负责起草大纲标题。
*   `write_report_introduction()` 和 `write_conclusion()`：负责写头和尾。

### 阶段二：最核心的 `generate_report` 方法
这个函数是真正的“执笔人”，你会看到很多非常工程化的细节：
```python
# gpt_researcher/actions/report_generation.py 节选
async def generate_report(query, context, agent_role_prompt, report_type, ...):
    
    # 细节 1：针对性注入。如果是“子章节报告”，Prompt 会非常特殊，它需要知道当前写到哪了，避免和前面写过的内容重复。
    if report_type == "subtopic_report":
        content = f"{generate_prompt(query, existing_headers, relevant_written_contents, ...)}"
    else:
        content = f"{generate_prompt(query, context, report_source, ...)}"
    
    # 细节 2：图文并茂 (Multi-modal Assembly)
    # 如果系统提前通过工具生成或找到了相关图片，它会在这里使用极强的 Prompt 约束 LLM，强制它在输出文本时把图片的 Markdown 链接嵌入到合适的位置。
    if available_images:
        images_info = "\n".join([f"- Image..."] )
        content += f"\nAVAILABLE IMAGES: You have the following pre-generated images available. Embed them..."

    # 细节 3：超强降级容错！(Fallback Mechanism)
    try:
        # 标准的 Chat 接口，分为 system prompt (人设) 和 user prompt (任务)
        report = await create_chat_completion(
            messages=[
                {"role": "system", "content": f"{agent_role_prompt}"},
                {"role": "user", "content": content},
            ], ...)
    except Exception:
        try:
            # 💡 神来之笔：如果报错了怎么办？
            # 有些开源大模型（如早期版本的某些国产模型）对 System Role 支持极差甚至会报错。
            # 这里立刻降级，把 system 和 user 的内容硬拼接成一段话，全部塞进 user 角色里再试一次！
            report = await create_chat_completion(
                messages=[
                    {"role": "user", "content": f"{agent_role_prompt}\n\n{content}"},
                ], ...)
```

这种处理方式，保证了无论你接的是 OpenAI 还是各种奇奇怪怪的本地小模型，报告都能尽量生成出来。

## 3. 实操体验：模拟长文组装车间 (MRE)

让我们写一个简短的脚本，体验一下“先出大纲、再写小节”的流水线。

### MRE 代码：长文分拣器
在根目录下新建文件 `mre_module6.py`：

```python
import asyncio
from gpt_researcher.config.config import Config
from gpt_researcher.actions.report_generation import generate_draft_section_titles, generate_report
from gpt_researcher.prompts import PromptFamily

async def main():
    print("🏭 万字长文组装车间启动...\n")
    
    cfg = Config()
    prompt_family = PromptFamily()
    
    query = "2024年全球新能源汽车出海现状分析"
    context = "这里假装是一大堆从网页上爬下来并经过 RAG 压缩的关于新能源车出海的数据..."
    role = "你是一个资深的汽车行业分析师。"
    
    # 1. 生成大纲
    print("⏳ 步骤一：正在构思报告大纲...")
    titles = await generate_draft_section_titles(
        query=query,
        current_subtopic=query,
        context=context,
        role=role,
        config=cfg,
        prompt_family=prompt_family
    )
    
    print("\n✅ 生成大纲如下：")
    for i, title in enumerate(titles):
        # 简单过滤空行
        if title.strip():
            print(f"  {i+1}. {title}")
            
    # 我们只拿第一节来做演示
    if titles:
        first_section = titles[0]
        print(f"\n⏳ 步骤二：正在单独撰写第一小节【{first_section}】...")
        
        # 2. 调用 generate_report 生成单独的小节
        section_content = await generate_report(
            query=first_section,        # 现在的任务不是写全文，而是写这一个小节
            context=context,
            agent_role_prompt=role,
            report_type="subtopic_report", # 明确告诉它这是子章节
            tone="Objective",
            report_source="web",
            websocket=None,
            cfg=cfg,
            main_topic=query,
            existing_headers=[],        # 传入之前已有的标题，防止大模型重复写
            relevant_written_contents=[], 
            prompt_family=prompt_family
        )
        
        print(f"\n✅ 第一小节撰写完毕，内容如下：\n{'-'*50}\n{section_content[:300]}...\n{'-'*50}")
        print("\n💡 真实流程中，系统会并发或循环生成所有小节，最后拼接引言和结论，再使用 Markdown 转换为 PDF/Docx！")

if __name__ == "__main__":
    asyncio.run(main())
```

### 运行结果预期
运行后（`python mre_module6.py`），你会清晰地看到它先列出了比如“一、市场概述；二、核心品牌出海策略；三、面临的关税阻力”等提纲。然后，大模型会死死盯住“市场概述”这一个点，利用传给它的 Context，写出一份几百字的高质量段落。

这就是**让大模型干大工程的核心秘诀：拆！**
