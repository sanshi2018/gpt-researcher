## 2.2 Memory 类 — Embedding 提供商管理

**源码位置**：`gpt_researcher/memory/embeddings.py`

### 设计理念

Memory 类本质上是一个 **Embedding 工厂**，通过 `match-case` 模式匹配，支持 20+ Embedding 提供商的懒加载初始化。

### 核心代码结构

```python
class Memory:
    """管理文档相似度和检索的 Embedding 生成"""
    
    def __init__(self, embedding_provider: str, model: str, **embedding_kwargs):
        _embeddings = None
        match embedding_provider:
            case "openai":
                from langchain_openai import OpenAIEmbeddings
                _embeddings = OpenAIEmbeddings(model=model, **embedding_kwargs)
            case "azure_openai":
                from langchain_openai import AzureOpenAIEmbeddings
                _embeddings = AzureOpenAIEmbeddings(...)
            case "ollama":
                from langchain_ollama import OllamaEmbeddings
                _embeddings = OllamaEmbeddings(model=model, base_url=...)
            # ... 20+ 提供商
            case _:
                raise Exception("Embedding not found.")
        
        self._embeddings = _embeddings
    
    def get_embeddings(self):
        """返回配置好的 LangChain Embeddings 实例"""
        return self._embeddings
```

### 支持的 Embedding 提供商

| 提供商 | 类型 | 备注 |
|--------|------|------|
| `openai` | 云端 | text-embedding-3-small/large |
| `azure_openai` | 云端 | Azure 部署 |
| `cohere` | 云端 | CohereEmbeddings |
| `google_genai` | 云端 | Google Generative AI |
| `google_vertexai` | 云端 | Vertex AI |
| `ollama` | 本地 | 需要 OLLAMA_BASE_URL |
| `huggingface` | 本地 | sentence-transformers |
| `voyageai` | 云端 | 需要 VOYAGE_API_KEY |
| `dashscope` | 云端 | 阿里云 |
| `bedrock` | 云端 | AWS Bedrock |
| `custom` | 自定义 | 兼容 OpenAI API 的服务 |
| `openrouter` | 云端 | OpenRouter 路由 |
| `minimax` | 云端 | MiniMax API |

### 关键设计决策

1. **懒加载 (Lazy Import)**：每个提供商的依赖库只在需要时才导入，避免安装所有依赖
2. **统一接口**：所有提供商都返回 LangChain 的 `Embeddings` 接口，下游代码无需关心具体实现
3. **环境变量驱动**：API Key 等敏感信息通过环境变量注入

### 在 GPTResearcher 中的初始化

```python
# agent.py 第 176-178 行
self.memory = Memory(
    self.cfg.embedding_provider,    # 从 Config 解析出的提供商名
    self.cfg.embedding_model,       # 模型名称
    **self.cfg.embedding_kwargs     # 额外参数
)
```

### Embedding 配置格式

Config 中的 `EMBEDDING` 使用 `provider:model` 格式：

```bash
EMBEDDING=openai:text-embedding-3-small      # OpenAI
EMBEDDING=ollama:nomic-embed-text             # Ollama 本地
EMBEDDING=huggingface:sentence-transformers/all-MiniLM-L6-v2  # HuggingFace
```

## 2.3 VectorStoreWrapper — 向量存储封装

**源码位置**：`gpt_researcher/vector_store/vector_store.py`

### 设计理念

`VectorStoreWrapper` 是对 LangChain `VectorStore` 的轻量级封装，负责：
1. 将 GPT-Researcher 文档格式转换为 LangChain Document
2. 自动分块 (Chunking)
3. 提供异步相似度搜索

### 核心代码

```python
class VectorStoreWrapper:
    def __init__(self, vector_store: VectorStore):
        self.vector_store = vector_store

    def load(self, documents):
        """将文档加载到向量库中"""
        # Step 1: GPT-Researcher Dict → LangChain Document
        langchain_documents = self._create_langchain_documents(documents)
        # Step 2: 分块
        splitted_documents = self._split_documents(langchain_documents)
        # Step 3: 入库
        self.vector_store.add_documents(splitted_documents)
    
    def _create_langchain_documents(self, data):
        """格式转换: {raw_content, url} → Document(page_content, metadata)"""
        return [
            Document(
                page_content=item["raw_content"], 
                metadata={"source": item["url"]}
            ) for item in data
        ]
    
    def _split_documents(self, documents, chunk_size=1000, chunk_overlap=200):
        """RecursiveCharacterTextSplitter 分块"""
        text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=chunk_size,
            chunk_overlap=chunk_overlap,
        )
        return text_splitter.split_documents(documents)

    async def asimilarity_search(self, query, k, filter):
        """异步相似度搜索"""
        return await self.vector_store.asimilarity_search(
            query=query, k=k, filter=filter
        )
```

### 使用场景

VectorStore 是 **可选** 组件。当用户提供自定义向量库时使用：

```python
# 用户代码
from langchain_chroma import Chroma
vector_store = Chroma(embedding_function=embeddings)

researcher = GPTResearcher(
    query="...",
    vector_store=vector_store,           # 传入自定义向量库
    vector_store_filter={"type": "pdf"}, # 可选过滤器
    report_source="langchain_vectorstore"
)
```

### 数据流

```
原始文档 (dict)
  ↓ _create_langchain_documents()
LangChain Document
  ↓ _split_documents() [chunk_size=1000, overlap=200]
分块后的 Documents
  ↓ vector_store.add_documents()
向量库 (Chroma/FAISS/Pinecone/...)
  ↓ asimilarity_search(query, k=5)
相似文档列表
```

## 2.4 多Agent 状态管理

**源码位置**：`multi_agents/memory/research.py` 和 `multi_agents/memory/draft.py`

### ResearchState — 研究全局状态

```python
class ResearchState(TypedDict):
    task: dict                    # 研究任务配置
    initial_research: str         # 初步研究结果
    sections: List[str]           # 计划的章节列表
    research_data: List[dict]     # 各章节的研究数据
    human_feedback: str           # 人工反馈
    # 报告布局
    title: str                    # 报告标题
    headers: dict                 # 报告头信息
    date: str                     # 日期
    table_of_contents: str        # 目录
    introduction: str             # 引言
    conclusion: str               # 结论
    sources: List[str]            # 来源列表
    report: str                   # 最终报告
```

### DraftState — 草稿子状态

```python
class DraftState(TypedDict):
    task: dict                    # 任务配置
    topic: str                    # 当前子主题
    draft: dict                   # 草稿内容
    review: str                   # 审查反馈
    revision_notes: str           # 修订笔记
```

### 状态在 LangGraph 中的流转

```
ResearchState 用于主工作流：
browser → planner → human → researcher → writer → publisher

DraftState 用于每个章节的子工作流：
researcher → reviewer → reviser → (循环直到 review 为 None)
```

### 关键设计洞察

1. **TypedDict 而非 Pydantic**：使用 TypedDict 定义状态，与 LangGraph 原生兼容
2. **分层状态**：全局 ResearchState + 子流程 DraftState，避免状态膨胀
3. **human_feedback 字段**：支持 Human-in-the-Loop，人类可以在规划阶段介入

## 2.5 GPTResearcher 中的隐式记忆

除了显式的 Memory 和 VectorStore，GPTResearcher 还通过实例变量维护 **隐式记忆**：

```python
class GPTResearcher:
    def __init__(self, ...):
        self.context = []              # 累积的研究上下文（核心记忆）
        self.visited_urls = set()      # 已访问URL集合（避免重复抓取）
        self.research_sources = []     # 研究来源列表
        self.research_images = []      # 收集的图片
        self.research_costs = 0.0      # API费用累计
        self.step_costs = {}           # 分步骤费用
```

### context 的生命周期

```
初始化: self.context = []
  ↓ conduct_research()
积累: self.context = research_conductor.conduct_research()
  ↓ 多个子查询并行执行，结果合并
压缩: 通过 ContextCompressor 过滤相关内容
  ↓ write_report()
消费: 作为 LLM 上下文生成报告
```

### visited_urls 的共享机制

在 Deep Research 中，visited_urls 在父子 Researcher 之间共享，避免重复抓取：

```python
# deep_research.py 中创建子 Researcher
researcher = GPTResearcher(
    query=serp_query['query'],
    visited_urls=self.visited_urls,  # 共享已访问URL集合
    ...
)
```

context_compressor = ContextCompressor(
