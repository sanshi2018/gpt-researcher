## 一、项目全局概览

GPT-Researcher 是一个**自主研究代理系统**，其核心理念是：给定一个研究问题，系统自动完成"搜索 → 抓取 → 压缩 → 生成报告"的全流程。项目包含两套独立但互补的架构：

| 架构层级 | 入口 | 核心类 | 编排方式 |
|---------|------|--------|---------|
| **单体 Agent** | `gpt_researcher/agent.py` | `GPTResearcher` | 纯 Python async 编排 |
| **多 Agent 协作** | `multi_agents/` | `ChiefEditorAgent` | LangGraph `StateGraph` |

### 1.1 单体 Agent 核心运作流

```
用户查询 (query)
    │
    ▼
┌─────────────────────┐
│  choose_agent()     │  ← LLM 自动选择最佳研究角色
│  (agent_creator.py) │
└────────┬────────────┘
         │
         ▼
┌─────────────────────────────────┐
│  ResearchConductor.conduct_research()  │
│  (skills/researcher.py)                │
│                                        │
│  1. plan_research() → 生成子查询       │
│  2. 并发执行 _process_sub_query()      │
│     ├─ _scrape_data_by_urls()          │
│     │   ├─ Retriever.search()          │
│     │   └─ BrowserManager.browse()     │
│     └─ ContextManager                  │
│        .get_similar_content_by_query() │
│        └─ ContextCompressor           │
│           ├─ TextSplitter              │
│           └─ EmbeddingsFilter          │
│  3. SourceCurator.curate_sources()     │
└────────┬────────────────────────────────┘
         │
         ▼
┌─────────────────────────────┐
│  ReportGenerator.write_report()  │
│  (skills/writer.py)              │
│  → generate_report()            │
│    (actions/report_generation.py)│
└──────────────────────────────────┘
         │
         ▼
     最终研究报告 (Markdown)
```

```
                    ┌──────────┐
                    │  START   │
                    └────┬─────┘
                         │
                         ▼
              ┌─────────────────────┐
              │  browser (初始研究)  │  ← ResearchAgent.run_initial_research()
              └────────┬────────────┘
                       │
                       ▼
              ┌─────────────────────┐
              │  planner (规划大纲) │  ← EditorAgent.plan_research()

                       │

            │

        ┌───────┐
```

### 📘 模块 8：深度研究与 MCP 工具调用策略

**文件**: `08_深度研究与MCP工具调用策略.md`

**聚焦问题**: Deep Research 的递归研究树如何工作？MCP 工具调用的三种策略有何区别？

**内容概要**:
- **DeepResearchSkill 递归研究机制**：
  - `run()` 入口：生成研究计划 → 组合查询 → 递归深研
  - `deep_research()` 的递归树：`depth` 层 × `breadth` 分支
  - `breadth` 递减策略：`new_breadth = max(2, breadth // 2)`
  - `Semaphore` 并发控制：`concurrency_limit` 限制同时运行的子研究
  - `process_research_results()` 的 Learning 提取与 Citation 追踪

## 四、核心架构设计哲学总结

### 4.1 分层解耦设计

```
┌────────────────────────────────────────────┐
│            多 Agent 协作层                  │
│  (multi_agents/ - LangGraph StateGraph)    │
├────────────────────────────────────────────┤
│            单体 Agent 引擎层               │
│  (gpt_researcher/agent.py - GPTResearcher) │
├────────────────────────────────────────────┤
│            技能层 (Skills)                  │
│  ResearchConductor │ ReportGenerator       │
│  ContextManager    │ SourceCurator         │
│  DeepResearchSkill │ BrowserManager        │
├────────────────────────────────────────────┤
│            动作层 (Actions)                 │
│  choose_agent │ generate_sub_queries       │
│  generate_report │ get_retrievers          │
├────────────────────────────────────────────┤
│            基础设施层                       │
│  LLM Provider │ Memory/Embeddings          │
│  Retrievers   │ Scraper │ VectorStore      │
└────────────────────────────────────────────┘
```

### 4.2 关键设计模式

| 设计模式 | 应用位置 | 说明 |
|---------|---------|------|
| **工厂模式** | `get_retriever()`, `GenericLLMProvider.from_provider()` | 通过字符串名称动态实例化 |
| **策略模式** | MCP Strategy (`fast`/`deep`/`disabled`) | 运行时切换 MCP 调用策略 |
| **组合模式** | `GPTResearcher` 组合 5 个 Skill 对象 | 通过组合而非继承扩展能力 |
| **观察者模式** | `cost_callback`, `log_handler`, `websocket` | 事件驱动的成本追踪与日志 |
| **模板方法** | `PromptFamily` | 可替换的 Prompt 模板族 |
| **状态机** | `ResearchState` / `DraftState` (TypedDict) | LangGraph 节点间的状态流转 |

### 4.3 三级 LLM 模型分层

```
strategic_llm  ──→ 用于「规划」：生成子查询、研究大纲
    │                  (支持 reasoning_effort 调节)
    ▼
smart_llm      ──→ 用于「生成」：撰写报告、选择角色、来源评估
    │
    ▼
fast_llm       ──→ 用于「轻量任务」：快速摘要等
```
