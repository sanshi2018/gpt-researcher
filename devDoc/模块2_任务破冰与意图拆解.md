# 模块二：任务破冰与意图拆解 (Query Processing & Task Breakdown)

在模块一中，我们见识了“全能主编”如何发号施令。但你有没有想过，当用户扔过来一个非常宏大甚至模糊的问题时，比如：“帮我调研一下 2024 年固态电池的商业化进展和主要阻力”，Agent 是怎么知道该去搜什么的？

如果直接把这句话作为关键词扔给 Google，搜出来的可能都是些不知所云的公关稿。

优秀的 Agent 必须具备**“任务破冰与意图拆解”**的能力。它需要像一个经验丰富的侦探，把大案子拆解成一个个确定的线索，分头去查。这也就是我们要剖析的 `actions/query_processing.py` 模块。

---

## 1. 核心机制：大处着眼，小处下手

GPT-Researcher 解决这个问题的思路非常直白：**大模型驱动的 Query 生成**。

它专门向 LLM 发送一条指令（Prompt），大意是：“你现在是一个专业的资料搜集员。用户现在的总体目标是 X，请你帮我想出 3 到 4 个最容易在搜索引擎上找到干货的搜索关键词（Sub-queries）。”

这就把一个“开放式问题”，转化成了多个“精准定位”的搜索动作。

---

## 2. 源码深层解剖：`query_processing.py`

打开 `gpt_researcher/actions/query_processing.py`，核心逻辑在 `generate_sub_queries` 这个函数中：

### 第一步：获取专业的 Prompt
系统会调用 `PromptFamily.generate_search_queries_prompt` 来生成对话提示词：
```python
# gpt_researcher/prompts.py
def generate_search_queries_prompt(question, max_iterations=3, context=[]):
    return f"""
    Write {max_iterations} google search queries to search online that form an objective opinion from the following task: "{task}"
    ...
    You must respond with a list of strings in the following format: ["query 1", "query 2", "query 3"].
    The response should contain ONLY the list.
    """
```
**看点**：这里直接通过 Prompt 约束了 LLM 输出的格式——必须是一个 JSON 格式的字符串列表。这就是最原始、最朴素的 **Structured Output (结构化输出)**。

### 第二步：带容错机制的模型调用 (Fallback & Self-Correction)
在向模型发起请求时，源码中做了一层非常优雅的**降级容错机制**：
```python
# gpt_researcher/actions/query_processing.py 节选
try:
    # 尝试用“战略模型”(Strategic LLM，比如 GPT-4o 或 Claude-3.5-Sonnet) 进行高级推理
    response = await create_chat_completion(model=cfg.strategic_llm_model, ...)
except Exception as e:
    # 如果战略模型出错了（比如上下文超限或限流），尝试减小 Token 限制重试
    try:
        response = await create_chat_completion(max_tokens=cfg.strategic_token_limit, ...)
    except Exception as e:
        # 如果还是不行，那就降级回退到“智能模型”(Smart LLM，比如 GPT-4o-mini 或 Haiku) 保底！
        response = await create_chat_completion(model=cfg.smart_llm_model, ...)
```
**类比**：这就像是公司遇到了难题，先请“首席战略官”来解决；如果他忙不过来（报错），就让他换个简单点的方式再试一次；如果还不行，就交给底下的“高级经理”来兜底完成。

### 第三步：野蛮解析 `json_repair`
大模型是很任性的，即使你告诉它“ONLY the list”，它可能还是会给你回一句：“*好的，为您生成以下搜索词：["词1", "词2"]*”。

如果你直接用 Python 自带的 `json.loads`，程序当场崩溃。
因此，GPT-Researcher 使用了 `json_repair.loads(response)`。这个库利用正则和修正算法，硬生生地从大模型的废话中抠出那个 `[]` 数组，极大地增强了系统的鲁棒性。

## 3. 实操体验：拆解任务的最小示例 (MRE)

让我们写一个极简的脚本，抛开 Researcher 的外壳，直接调用这个“破冰工具”，看看它是怎么把自然语言变成搜索词的。

### MRE 代码：智能子任务拆解器
在根目录下新建文件 `mre_module2.py`：

```python
import asyncio
from gpt_researcher.config import Config
from gpt_researcher.prompts import PromptFamily
from gpt_researcher.utils.llm import create_chat_completion
import json_repair

async def main():
    # 1. 模拟用户的一个宏大问题
    user_query = "调研 2024 年人形机器人的技术瓶颈与各家大厂的布局策略"
    print(f"🔥 原始用户问题: {user_query}\n")

    # 2. 初始化配置 (会自动读取 .env 中的大模型配置)
    cfg = Config()

    # 3. 拿到专门用于生成子问题的 Prompt
    prompt = PromptFamily.generate_search_queries_prompt(
        question=user_query,
        parent_query="",
        report_type="research_report",
        max_iterations=4 # 我们要求拆解出 4 个搜索词
    )
    print("🧠 发送给 LLM 的指令 (Prompt):")
    print(prompt)
    print("-" * 50)

    # 4. 调用大模型进行推理拆解
    print("⏳ 正在思考并拆解任务...\n")
    response_text = await create_chat_completion(
        model=cfg.smart_llm_model,          # 使用默认的 Smart LLM
        messages=[{"role": "user", "content": prompt}],
        llm_provider=cfg.smart_llm_provider,
        llm_kwargs=cfg.llm_kwargs
    )
    
    print(f"🤖 大模型原始返回文本:\n{response_text}\n")

    # 5. 使用 json_repair 提纯数据
    sub_queries = json_repair.loads(response_text)
    
    print("✅ 最终提纯后的搜索子任务数组:")
    for i, q in enumerate(sub_queries, 1):
        print(f"  [子任务 {i}] -> {q}")

if __name__ == "__main__":
    asyncio.run(main())
```

### 运行结果预期
运行这个脚本后，你会看到它成功将那句复杂的话，拆成了类似这样的 4 个 Google 搜索词：
1. `2024 人形机器人 技术瓶颈 核心难点`
2. `特斯拉 优必选 波士顿动力 人形机器人 布局对比`
3. `2024 AI 具身智能 人形机器人 进展`
4. `人形机器人 商业化落地 挑战 2024`

这就是 Agent 走向自主行动的**第一步：自己给自己派发工单！**
