# 文档生成要求

## 文档规划原则
- 按照项目逻辑模块拆分，共生成 7～10 个 Markdown 文件
- 编号格式：00_overview.md、01_xxx.md 依此类推
- 每个文档覆盖一个内聚的技术模块，不重叠，不遗漏
- 内容层次：架构图/流程说明 → 核心源码解析 → 技术原理 → 实操示例

## 必须覆盖的技术重点（每个模块按相关性选取）

### Agent 与多 Agent 系统
- 单 Agent vs 多 Agent 架构对比
- Agent 角色划分（Researcher / Editor / Publisher 等）
- Agent 间通信与协调机制
- ReAct / Plan-and-Execute 等 Agent 范式

### Function Call 与工具调用
- OpenAI Function Calling 规范与 JSON Schema 定义
- 工具注册、路由、执行闭环
- 错误处理与重试机制（含 RunnableRetry）
- Structured Output 与 Pydantic 模型校验

### 记忆与上下文管理
- 短期记忆（对话轮次内）vs 长期记忆（跨会话持久化）
- 上下文窗口管理：截断策略、摘要压缩
- 状态机设计（LangGraph State / StateGraph）
- Token 预算管理

### RAG 完整链路
- 查询理解与改写（Query Rewriting / HyDE）
- 检索策略：Dense / Sparse / Hybrid 检索
- 召回（Recall）与相关性过滤
- 重排序（Reranking）：Cross-Encoder、Cohere Rerank
- 文档分块（Chunking）策略：固定长度、递归、语义分块
- 上下文压缩（Contextual Compression）

### VectorDB 与向量检索
- Embedding 模型选型与比较
- VectorDB 对比（Chroma / Pinecone / Qdrant / FAISS）
- 索引构建、相似度计算、ANN 算法
- 元数据过滤与混合查询

### LangGraph / LangChain 工作流
- Graph 构建：Node / Edge / Conditional Edge
- StateGraph 状态流转与持久化 Checkpoint
- 流式输出（Streaming）
- 并行分支与 Map-Reduce 模式
- Human-in-the-loop 与中断机制

### 爬取、搜索与内容处理
- 搜索引擎 API 集成（Tavily / Serper / Bing）
- 网页爬取与内容提取（BeautifulSoup / Playwright）
- 内容清洗、去重、质量过滤

### MCP（Model Context Protocol）
- MCP 架构与 Server / Client 设计
- Tool / Resource / Prompt 三类原语
- 与 Function Calling 的异同
- 在 gpt-researcher 中的集成方式（如有）

### 提示词工程
- System Prompt 设计模式
- Few-shot / Chain-of-Thought / Self-Consistency
- 输出格式控制（JSON Mode / XML 结构化）
- 动态提示词拼装

### 可观测性与评估
- LangSmith Tracing 集成
- 评估指标：Faithfulness / Relevance / Groundedness
- 日志、成本追踪、性能监控

### benchmark
- RAGAS 集成以及完整评估流程的组装
- 基于真实故障分析的定制标准选择
- LLM 作为裁判的完整度指标，包括朴素方法与高级方法
- 基于 RAGAS 的幻觉检测，用于事实准确性
- 端到端管道组装，将所有指标整合为一个统一分数

### 补充技术点（请按项目实际情况覆盖）
- Async 异步并发架构（asyncio / aiohttp）
- 配置管理与环境变量注入
- 报告生成与 Markdown 渲染
- WebSocket / SSE 实时推送
- Docker / 部署架构
